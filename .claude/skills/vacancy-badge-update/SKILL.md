# 空室バッジ更新

空室数（個室の残り部屋数）が変わるたびに、**サイト内6箇所すべて**を同じ内容に揃える定型業務。1箇所でも更新漏れがあると空室数の表記が食い違うので、必ず全箇所をチェックする。

## 更新対象（全6箇所）

| ファイル | 施設 | 種類 | 目印 |
|---|---|---|---|
| `index.html` | うらら | トップの施設カード内バッジ | `vacancy-badge` クラスの span（うららカード側） |
| `index.html` | セラヴィ豊岡 | トップの施設カード内バッジ | `vacancy-badge` クラスの span（セラヴィ豊岡カード側） |
| `urara.html` | うらら | ページ上部ヒーローのバッジ | `vacancy-badge` クラスの span |
| `urara.html` | うらら | 施設概要テーブルの「空き状況」行 | `<tr><th>空き状況</th><td>...` |
| `celavie.html` | セラヴィ豊岡 | ページ上部ヒーローのバッジ | `vacancy-badge` クラスの span |
| `celavie.html` | セラヴィ豊岡 | 施設概要テーブルの「空き状況」行 | `<tr><th>空き状況</th><td>...` |

`vacancy-badge` と「空き状況」でファイル内をgrepすれば該当箇所を確実に見つけられる。

## 手順

1. ユーザーから新しい空室数（施設ごと）をヒアリングする。
2. 上記6箇所を**全て**grepで洗い出し、該当施設の箇所だけを新しい部屋数に書き換える。
   - バッジの文言パターン例：`✅ 現在、個室に空きがございます（残りN部屋）` または `✅ 空室情報：個室 残りN部屋`（既存の文言スタイルを踏襲し、新しい言い回しを作らない）。
   - 空きが0になった場合は「満室」等、既存にない文言が必要になるので、その場合はユーザーに表記案を確認してから進める。
   - 概要テーブルの「空き状況」行にある日付（`※YYYY-MM-DD時点`）も更新日に合わせて書き換える。
   - 各バッジ直前のHTMLコメント（`<!-- 空室状況は変動するため定期的に手動更新（YYYY-MM-DD時点） -->`）の日付も更新する。
3. 絵文字は [[feedback-emoji-rendering]]（自動メモリー）のルールに従う。✅は問題ないが、🈳等の囲み文字系絵文字は使わない。
4. 更新後、`git diff`で6箇所すべてが意図通り変わっているか確認する。
5. コミットメッセージは `fix: 空室バッジを更新（施設名 残りN部屋→M部屋）` のような形式にする（過去のコミット履歴の書式を踏襲）。
6. 完了後、自動メモリーの `project-overview.md` に最新の空室数を反映し、CLAUDE.mdの取り決めに従ってObsidian `業務\きぼうオンライン (1)\` にもミラーする。
