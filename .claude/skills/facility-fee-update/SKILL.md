# 施設料金更新

料金改定時に、**料金を掲載している3箇所すべて**を同じ数値に揃える定型業務。1箇所でも古い金額が残ると、シミュレーター結果とページ表記が食い違う。

## 更新対象（全3箇所）

| ファイル | 内容 |
|---|---|
| `simulation.html` | `FACILITIES` というJSオブジェクト（施設ごとの居室タイプ・月額・暖房費・入居時費用などの実データ） |
| `urara.html` | 施設概要テーブルの「月額費用目安」行（`<tr><th>月額費用目安</th><td>...`） |
| `celavie.html` | 施設概要テーブルの「月額費用目安」行（`<tr><th>月額費用目安</th><td>...`） |

現在の実料金データは自動メモリー [[fee-simulator]] にも記録されている（うらら／セラヴィ豊岡それぞれの居室タイプ別金額・入居時費用・暖房費）。改定前にまずこのメモリーで現状値を確認し、齟齬がないか照合する。

## 手順

1. ユーザーから改定内容（どの施設・どの居室タイプ・新しい金額）をヒアリングする。
2. `simulation.html` の `FACILITIES` オブジェクト内、該当施設・該当居室タイプの数値を書き換える。
3. `urara.html` / `celavie.html` の該当する施設概要テーブル「月額費用目安」行の実数値表記を同じ内容に書き換える（居室タイプごとの幅表記〈¥◯〜¥◯〉の書式を踏襲）。
4. 介護保険自己負担の計算方式（区分支給限度基準額×自己負担割合の簡易方式）や夫婦室・2人部屋の人別計算ロジック（`renderCarePersons`等）自体は料金改定では触らない。金額データのみを更新する。
5. 更新後、`git diff`で3箇所すべてが一致した内容に変わっているか確認する。
6. コミットメッセージは `feat: 料金改定に伴い月額費用を更新（施設名・居室タイプ）` のような形式にする。
7. 完了後、自動メモリー [[fee-simulator]] の実料金データ部分を新しい金額に更新し、CLAUDE.mdの取り決めに従ってObsidian `業務\きぼうオンライン (1)\` にもミラーする。
